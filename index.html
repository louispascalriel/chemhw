ChemHW Setup
Connect your Supabase database — takes about 5 minutes
1
Create a free Supabase project
Go to supabase.com, sign up, and create a new project. Wait for it to finish provisioning (~2 min).
2
Run this SQL in the Supabase SQL editor
Go to Project → SQL Editor and paste + run this: 
create table if not exists profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  username text unique not null,
  role text not null default 'student',
  class_code text,
  email text,
  created_at timestamptz default now()
);
create table if not exists student_results (
  id uuid primary key default gen_random_uuid(),
  student_id uuid references profiles(id) on delete cascade,
  question_id int not null,
  question_text text,
  correct boolean not null,
  attempts_used int not null default 1,
  student_answer text,
  answered_at timestamptz default now()
);
alter table profiles enable row level security;
alter table student_results enable row level security;
create policy "Users read own profile" on profiles for select using (auth.uid() = id);
create policy "Users update own profile" on profiles for update using (auth.uid() = id);
create policy "Teachers read all profiles" on profiles for select using (true);
create policy "Users read own results" on student_results for select using (auth.uid() = student_id);
create policy "Users insert own results" on student_results for insert with check (auth.uid() = student_id);
create policy "Teachers read all results" on student_results for select using (true);
grant usage on schema public to anon, authenticated, service_role;
grant select, update on profiles to authenticated;
grant select on profiles to anon;
grant select, insert on student_results to authenticated;
grant select on student_results to service_role;
-- If upgrading from an older version, run these migration lines:
-- alter table student_results add column if not exists student_answer text;
-- alter table profiles add column if not exists email text;
3
Paste your Supabase credentials below
Find these under Project Settings → API.

If students see a "Could not save" error, your Supabase tables are missing role grants. Run this in your Supabase SQL Editor:

grant usage on schema public to anon, authenticated, service_role;
grant select, update on profiles to authenticated;
grant select on profiles to anon;
grant select, insert on student_results to authenticated;
grant select on student_results to service_role;
Admin access

Add your Supabase Service Role Key so the teacher dashboard can read all student results. Find it under Project Settings → API → service_role.

Keep this key private — do not share the deployed URL with students while this key is stored in browser settings.

Deployment

To share this with students, upload chemistry-hw.html to Vercel or Netlify — both are free and take under 2 minutes. Students visit the URL, register with your class code, and their results are saved to Supabase automatically.

Import questions

Fill in a spreadsheet (Excel or Google Sheets) and import it as a .csv file. Questions will be added to your existing bank — nothing will be overwritten.

Column What to enter
typeformula, mc, sci, or calc
assignmente.g. HW 1 (any name you like)
categoryHW or ICE
textThe question
answersFormula only — separate multiple accepted answers with a semicolon (e.g. H2O;HOH)
option_a … option_dMC only — one column per answer choice
correct_optionMC only — letter of correct choice (A, B, C, or D)
valueSci / Calc only — the correct numeric answer (e.g. 6.022e23)
unitCalc only — e.g. g/mol (optional)
tolerance_pctSci / Calc only — % margin e.g. 2 means ±2% (optional)
hintOptional hint shown to students
